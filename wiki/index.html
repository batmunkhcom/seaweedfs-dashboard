Cluster & Topology

Full topology tree (DC → Rack → Node), master/volume/filer server tables, leader election info, per-node volume limits, search/filter/sort.

Volume Management

Volume list with detail drawer, grow operation, vacuum/garbage collection trigger, collections CRUD, status indicators (ReadOnly, TTL, compaction).

Filer Browser

Directory navigation with breadcrumbs and pagination, file upload (drag-drop, multi-file, live progress), download, delete, create folders, metadata viewer.

S3 Management

Bucket CRUD, user management with access/secret key generation, policy editor with JSON validation, IAM sync to S3 gateway nodes via SSH. Gateway endpoints at .102:8333, .104:8333, .106:8333, .107:8333.

Backup & Restore

Filer metadata (LevelDB) backup via SSH/SFTP to dashboard server, restore process with filer restart, 30-day retention policy, API key authentication (bkp_ prefix).

Workers

Background job management — detect cluster work, execute volume balance/replication repair/garbage collect/filer repair/system check jobs. Monitor progress and review job history.

Disk Health

S.M.A.R.T. monitoring via SSH (smartctl --json), lifespan estimation (power-on hours, wear %, TBW), temperature and reallocated sector alerts.

Note: The dashboard backend acts as a BFF (Backend For Frontend) — it proxies and aggregates SeaweedFS API calls, handling multi-master failover transparently. The frontend never calls SeaweedFS directly.

Key Features

  • Multi-master failover — automatic switching with audit logging
  • Real-time monitoring — SSE push, no polling, auto-reconnect
  • Historical trends — configurable snapshot interval (default 60s), SQLite time-series storage
  • Smart alerts — configurable thresholds (disk > 90%, node offline, garbage > 0.5, readonly volumes)
  • Per-node volume limits — capacity planning with inline editing and growth enforcement
  • S.M.A.R.T. disk health — lifespan estimation, temperature/wear/reallocated sector monitoring
  • IAM sync — auto-sync S3 users/policies to gateway nodes via SSH
  • Structured logging — JSON-format logs via structlog with full audit trail
