Cluster & Topology

The Cluster page provides a complete view of the SeaweedFS cluster infrastructure — from the high-level topology tree down to individual server details with health indicators, resource usage, and configuration.

Leader Election

SeaweedFS uses the Raft consensus algorithm to elect a master leader. The leader handles all write operations and coordinates cluster state changes.

dc03 Master Nodes

NodeIPPortRaft PortRole
Master 1 10.10.95.101 9333 19333 Follower
Master 2 10.10.95.103 9333 19333 Follower
Master 3 10.10.95.105 9333 19333 Leader

The dashboard's SeaweedFS client (seaweed_client.py) is multi-master aware. It detects the current leader and automatically routes requests there. If the leader becomes unavailable, it cycles through the master hosts list to find the new leader — logging every failover action with full context.

Failover Behavior

# All master hosts are tried in order on failure:
SEAWEEDFS_MASTER_HOSTS=10.10.95.101:9333,10.10.95.103:9333,10.10.95.105:9333

# Client logic:
# 1. Try current leader (cached)
# 2. On failure, try next host
# 3. Continue until one responds or all exhausted
# 4. Log every failover attempt with traceback

Node Limits Management

Each volume server has a configurable maximum volume count. This is critical for capacity planning — with 1.8 TB disks and 30 GB volumes, each node can hold approximately 60 volumes.

Capacity Calculation

Disk size:    1800 GB
Volume size:    30 GB
Approx. max:    ~60 volumes per node

Set limits conservatively:
  - 50-55 for standard operation
  - 60 for maximum utilization

Volume limits are displayed on each node's card in the cluster view. Click the limit value to edit it inline — no page reload needed. When a node reaches its configured limit, that node's card shows a warning and new volume grow operations are blocked for that specific node.

Growth enforcement: The dashboard automatically blocks volume grow on nodes that have reached their limit. You must either increase the limit or free up space before growing volumes on that node.
