Getting Started

Everything you need to install, configure, and deploy the SeaweedFS Dashboard. Follow the steps below to get up and running.

System Requirements

Minimum Requirements

ComponentVersionNotes
Operating SystemLinux (Ubuntu 20.04+, Debian 11+, RHEL 8+)Dashboard server
Python3.11+Backend runtime
Node.js20+Frontend build
Nginx1.18+Production reverse proxy / static serve
Memory512 MB minimum1 GB recommended
Disk100 MB (app only)+ extra for SQLite data
Standalone server: The dashboard runs on its own server (10.10.0.80), not on any SeaweedFS node. This keeps monitoring independent of the storage cluster.

Installation

1. Clone the Repository

git clone git@github.com:batmunkhcom/seaweedfs-dashboard.git
cd seaweed-dashboard

2. Backend Setup

cd backend
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
cp .env.example .env

Edit .env to match your cluster configuration (see Configuration section below).

3. Frontend Setup

cd frontend
npm install

Configuration

All infrastructure configuration lives in backend/.env. This is the single source of truth for hosts, ports, credentials, and feature flags.

Important: Environment variables are only for infrastructure config (hosts, ports, credentials). Operational settings (thresholds, limits, intervals) belong in the runtime_settings SQLite table and can be changed via the Settings UI without restarting the server.
Redis is optional. When REDIS_URL is not set, in-memory fallbacks handle rate limiting and caching. Redis is only needed if you want persistent sessions across server restarts or SSE pub/sub for multi-process deployments.

Starting the Dashboard

The project includes manage.sh — a unified command interface for all operations:

manage.sh Commands

CommandDescription
./manage.sh devDev mode — backend hot-reload + Vite HMR (ports 8000 + 5173)
./manage.sh upStart all — backend (bg) + Nginx on port 8081
./manage.sh stopStop all services
./manage.sh restartStop then up
./manage.sh buildBuild frontend for production
./manage.sh statusShow running services + health check
./manage.sh lintRun backend + frontend linters
./manage.sh testRun all tests
./manage.sh logsTail backend logs
./manage.sh infoShow ports, domain, project path

Development Mode

./manage.sh dev
# Backend: http://localhost:8000 (hot-reload)
# Frontend: http://localhost:5173 (HMR, proxies /api to :8000)

Production Mode

./manage.sh build   # Compile frontend static files
./manage.sh up       # Start backend + Nginx
# Access: http://<server>:8081

Production Deployment

Backend — systemd Service

Create /etc/systemd/system/seaweed-dashboard.service:

[Unit]
Description=SeaweedFS Dashboard Backend
After=network.target

[Service]
Type=simple
User=www-data
WorkingDirectory=/srv/seaweed-dashboard/backend
EnvironmentFile=/srv/seaweed-dashboard/backend/.env
ExecStart=/srv/seaweed-dashboard/backend/venv/bin/uvicorn app.main:app \
  --host 127.0.0.1 --port 8000 --workers 1
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
sudo systemctl daemon-reload
sudo systemctl enable seaweed-dashboard
sudo systemctl start seaweed-dashboard

Nginx — Reverse Proxy

Create /etc/nginx/sites-available/seaweed-dashboard:

server {
    listen 8081;
    root /srv/seaweed-dashboard/frontend/dist;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_buffering off;
        proxy_read_timeout 3600s;
    }
}
sudo ln -s /etc/nginx/sites-available/seaweed-dashboard /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx

Cloudflared Tunnel

For public access via seaweed.mbm.mn:

ingress:
  - hostname: seaweed.mbm.mn
    service: http://10.10.0.80:8081
  - hostname: s3.mbm.mn
    service: http://10.10.1.8:80
  - service: http_status:404
Port 8081 is the contract. Nginx always listens on port 8081. Cloudflared, proxy configs, and firewall rules all depend on this. Never change this port without updating all external dependencies.

Docker Deployment

docker compose up -d

Multi-stage build compiles the frontend to static files, then serves everything through Nginx with the API proxy configured.
